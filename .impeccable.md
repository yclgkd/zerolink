## Design Context

### Users
ZeroLink serves people who need to transmit passwords, API keys, and short sensitive messages without creating accounts or trusting the server with plaintext. The main usage contexts are operational handoff, incident response, infrastructure access sharing, and other high-trust moments where users need clarity more than novelty.

### Brand Personality
Security-first, calm, trustworthy. The interface should feel deliberate, professional, and technically credible rather than flashy or playful.

### Aesthetic Direction
Prefer a restrained security-tool aesthetic: muted surfaces, clear hierarchy, limited accent usage, and strong readability. Avoid cyberpunk, neon, glassy "AI-generated" effects, and decorative intensity that competes with trust.

### Design Principles
1. Put the primary task above supporting explanation.
2. Make security-critical guidance easy to scan, not dense walls of small text.
3. Use emphasis sparingly so important actions and warnings stand out immediately.
4. Favor calm, stable visual language over spectacle.
5. Preserve protocol clarity and safety cues without increasing cognitive load.
